## What it does

When Google Gemini or AI Studio generates an image, it adds two markers:

1. A **visible** ✦ badge in the bottom-right corner.
2. An **invisible** watermark called **SynthID** — a statistical pattern embedded in the pixels that identifies the image as AI-generated. SynthID survives most common edits (screenshotting, cropping, compression, rotation).

UnmarkAI removes the first and disrupts the second:

- **Visible badge**: detected automatically and painted out using one of two inpainting engines. The result is indistinguishable from the rest of the image in typical use.
- **SynthID**: disrupted via a two-layer pass — light Gaussian noise (σ ≈ 1.5, below the human just-noticeable-difference threshold) plus an internal JPEG round-trip (quality 88) that destroys frequency-domain fingerprints without visible quality loss. _Disruption is probabilistic, not guaranteed — see the [Privacy](#privacy) section._

Both steps happen on your own machine. Nothing is uploaded.

## Features

- **Auto-clean on save** — when you click Gemini's download button, UnmarkAI intercepts it, cleans the image, and saves the cleaned version in its place.
- **Manual scan** — open the popup on any Gemini page, scan for images currently rendered on the page, and clean them one-by-one or all at once.
- **Two engines** — fast algorithmic cleaning for flat backgrounds, or neural-network cleaning (LaMa) for photos and textures. Pick per session, change any time.
- **Before/after comparison** — every successful clean can be previewed with a draggable slider, so you can check the result before using it.
- **View original** — if the clean isn't quite right, download the unprocessed image as a fallback. Kept for 5 minutes.
- **Cancel any time** — both the Pro model download (~198 MB) and each individual clean can be cancelled.
- **Batch mode** — "Clean all" runs through every image on a page sequentially, with a live time estimate.
- **Daily counter** — see how many images you've cleaned today, plus an all-time total.
- **Light and dark themes** — follows your system preference automatically.
- **Keyboard accessible** — full focus ring support, ARIA labels on every control, reduced-motion support.
- **Private by design** — no telemetry, no analytics, no network calls except the one-time model download.

## Installation

> The extension is currently distributed as source. Chrome Web Store listing is planned.

### Install from source (manual)

1. **Download the latest release.** Grab the ZIP from the [Releases page](https://github.com/YOUR_REPO/releases) or clone the repository:
   ```bash
   git clone https://github.com/YOUR_REPO/unmarkai.git
   ```

2. **Open Chrome's extension page.** In Chrome or any Chromium-based browser (Edge, Brave, Arc, Vivaldi), navigate to:
   ```
   chrome://extensions
   ```

3. **Enable Developer mode.** Toggle the switch in the top-right corner of the extensions page.

4. **Load the extension.** Click **"Load unpacked"** and select the folder where you downloaded or cloned UnmarkAI.

5. **Pin the icon** (optional but recommended). Click Chrome's puzzle-piece icon in the toolbar, find UnmarkAI, and click the pin icon next to it.

You should now see the UnmarkAI ✦ icon in your toolbar. Click it to open the popup.

### Updating

If you installed via `git clone`, run `git pull` in the extension folder and reload the extension from `chrome://extensions` (click the circular reload icon on the UnmarkAI card).

## Getting started

### Your first clean

1. Open **[gemini.google.com](https://gemini.google.com)** or **[aistudio.google.com](https://aistudio.google.com)**.
2. Generate an image as you normally would.
3. Click the download button on the image.
4. UnmarkAI intercepts the download, cleans the image, and saves the cleaned version to your default Downloads folder.

That's it. The first clean uses the **Quick** engine by default, which is instant.

### Automatic mode vs manual mode

Open the popup to see the **Auto-clean** toggle:

- **Auto-clean on** (default) — UnmarkAI intercepts every Gemini download button press and cleans the image automatically. You don't have to think about it.
- **Auto-clean off** (manual mode) — Gemini's download button behaves normally (saves the watermarked image). To clean, open the UnmarkAI popup, click **Scan**, and choose individual images or use **Clean all**.

Manual mode is useful if you want to selectively clean only some of the images in a chat.

### Cleaning from the popup

Even with auto-clean on, the popup is the way to clean images that weren't downloaded (e.g., images Gemini shows you during a chat that you want to keep).

1. Click the UnmarkAI icon in the toolbar.
2. Click **Scan**. UnmarkAI finds every image large enough to be a Gemini generation on the current page.
3. Hover any thumbnail to reveal the **Clean** button. Click it.
4. When the clean completes:
   - The image saves to your Downloads folder.
   - A **Compare** button lets you see a before/after slider.
   - A **View original** button lets you also save the uncleaned version.

To clean every image at once, click **Clean all**.

### Right-click shortcut

On any Gemini image you can **right-click → "✦ UnmarkAI — Clean & Download"** to run a clean without opening the popup.

## The two engines: Quick vs Pro

UnmarkAI ships with two independent engines. Pick whichever fits the image you're cleaning.

| | **Quick** | **Pro** |
|---|---|---|
| **Speed** | Instant (< 1 s) | ~30–40 s per image |
| **Size** | Built-in, no download | 198 MB one-time download |
| **Algorithm** | PatchMatch inpainting + boundary smoothing | LaMa neural network (ONNX) |
| **Best for** | Flat backgrounds, screenshots, diagrams, UI mockups | Photos, landscapes, complex textures |
| **Weakness** | Can leave subtle artifacts on highly detailed backgrounds | Slow; needs disk space |
| **Runs offline?** | Yes, always | Yes, after the model is downloaded |

### Quick engine

Quick uses a classical inpainting algorithm called **PatchMatch**: for each pixel under the watermark, it finds the most visually similar patch elsewhere in the image and copies its texture. A boundary-smoothing pass (Gauss-Seidel relaxation) then blends the patched region with its surroundings.

It's effectively instant and works well for most backgrounds. For flat-color or gradient regions it's indistinguishable from the original. For highly textured regions (foliage, hair, fabric weave) it occasionally leaves a subtle smudge.

### Pro engine

Pro uses **LaMa** (Large Mask inpainting), a state-of-the-art neural model originally trained for general-purpose image repair. UnmarkAI runs it locally via ONNX Runtime — no cloud inference.

The model is ~198 MB. It downloads once on first use, is cached permanently in your browser's IndexedDB, and works offline thereafter. You can cancel the download at any time or wipe the cache from the popup.

Pro produces near-invisible results even on the most challenging backgrounds. The trade-off is speed — a single image takes 30–40 seconds on a modern laptop CPU (single-threaded WASM). Future versions may add WebGPU or multi-threaded WASM support for a 3–4× speedup.

### Switching between engines

The engine picker is at the top of the popup. Your choice is remembered across sessions. You can switch freely — the Pro model stays cached even if you use Quick.

## Privacy

**UnmarkAI processes every image entirely on your device.** Your images never leave your browser.

### What UnmarkAI sees

- The URL of the image you're downloading or cleaning (to fetch the image bytes into the extension).
- The pixel data of that image (in your browser's memory, for processing).
- Your settings and a counter of how many images you've cleaned (in Chrome's local storage).

### What UnmarkAI does not do

- It does **not** upload your images anywhere.
- It does **not** send any telemetry, analytics, or crash reports.
- It does **not** track which sites you visit — it only activates on `gemini.google.com` and `aistudio.google.com`.
- It does **not** read any other page content.
- It does **not** store images on disk. The cleaned file is saved via Chrome's download mechanism (to your Downloads folder), and intermediate blobs are discarded after processing.
- It does **not** share any data with third parties. There are no third parties.

### The one network request

The only outbound network request UnmarkAI ever makes is the **one-time model download** for Pro mode, fetched from GitHub Releases when you first enable Pro. This download:
- Is explicitly initiated by your clicking on the Pro engine.
- Can be cancelled at any time.
- Only happens once — the model is cached in IndexedDB after the first successful download.
- Can be removed at any time via **Retry** in the model-status panel.

See the full [privacy policy](https://github.com/mabdsh/lama_fp32/blob/main/PRIVACY.md) for the authoritative statement.

### A note on SynthID

UnmarkAI's SynthID disruption is **probabilistic, not guaranteed**. The combination of Gaussian noise and JPEG round-trip destroys most known frequency-domain watermarks with high probability, but Google may update SynthID at any time to use different signals. If you're using UnmarkAI for a use case where SynthID detection could matter, be aware that:

1. No tool can guarantee 100% disruption against an adversarial, evolving watermark.
2. Even if the current SynthID is disrupted, future Google detectors may use additional signals.
3. Visible elements of the image (composition, style, content) are unaffected by UnmarkAI — only the ✦ badge is removed and the SynthID fingerprint is disturbed.

We make no claim that cleaned images will evade any particular detection system.

## Troubleshooting

### "No images found" after clicking Scan

- Make sure you're on a Gemini or AI Studio tab.
- Generate an image first — Scan only finds images already rendered on the page.
- Very small images (under 256 × 256 px) are skipped intentionally.

### Pro model stuck at "Downloading… X%"

- Check your internet connection. The model is ~198 MB, large for slow connections.
- Click **Cancel** on the progress panel and try again. GitHub's CDN occasionally throttles anonymous downloads.
- If it consistently fails, the Pro engine works independently — Quick engine doesn't need any download.

### "Couldn't set up Pro mode"

The panel shows a short error and a **Show details** button for the full message.
- Most common cause: network interruption. Click **Retry**.
- If the error mentions "integrity check failed," delete the cache (Retry automatically handles this) and download again.
- If the error persists, please [report a bug](#reporting-bugs) with the full error text.

### Auto-clean didn't trigger when I clicked download

- Make sure the UnmarkAI icon in the toolbar is coloured (active), not gray. If it's gray, the master toggle is off — click the icon and flip the switch.
- Check that **Auto-clean** is on inside the popup.
- Try refreshing the Gemini page. If you installed UnmarkAI after the page was already loaded, the content script may not have injected — a refresh fixes this.

### Cleaned image has a visible artifact

- Switch to **Pro** mode and try again. Quick uses a classical algorithm that occasionally struggles on highly textured backgrounds.
- If Pro also produces an artifact, please [report a bug](#reporting-bugs) with the original image attached so we can improve the detector.

### The popup looks broken or styled oddly

- Make sure your browser is up to date (Chrome 110+ / equivalent Chromium version).
- Try right-clicking the toolbar icon and choosing **Manage extension** → **Reload**.

### Stats counter reset

- "Today" resets at local midnight, which is expected.
- "All time" should never reset on its own. If it does, please report it.

## How it works (for the curious)

UnmarkAI is a Manifest V3 Chrome extension. Its architecture uses four cooperating contexts to work around platform constraints:

- **Main-world content script** runs in Gemini's own JavaScript context to intercept the download button before Chrome sees it (necessary because content scripts can't override page prototypes).
- **Isolated-world content script** runs in a sandboxed context with Chrome API access. It fetches image bytes, runs the Quick engine, and coordinates with the background worker.
- **Background service worker** handles download bookkeeping, badge/icon state, and routes messages. Service workers can't run ONNX, so it delegates to…
- **Offscreen document** — a hidden DOM context specifically for Pro-mode inference. ONNX Runtime needs DOM APIs (URL.createObjectURL, WebGL) that service workers don't have.

### Detection

The ✦ badge is located by two independent signals:
1. **Color fingerprinting** — looking for lavender-gray pixels in the bottom-right corner with a blue bias typical of Gemini's watermark palette.
2. **Star-shape detection** — measuring luminance contrast in the four diagonal-vs-cardinal axes to find the characteristic four-fold symmetry of the ✦ glyph.

The two signals are OR-combined, giving a recall close to 100% on the Gemini watermark across a wide range of image types.

### Quick engine pipeline

1. Detect the watermark bounding box.
2. Sample border pixels for a per-pixel inverse-distance-weighted (IDW) background estimate.
3. Measure surface complexity — flat surfaces use the IDW estimate directly; textured surfaces trigger PatchMatch exemplar search to find visually similar patches elsewhere in the image.
4. Gauss-Seidel relaxation at the fill boundary eliminates any visible seam.
5. Soft alpha blend in a feather zone ensures no hard edges remain.

### Pro engine pipeline

1. Detect the watermark bounding box, expand it with 64 px context padding.
2. Extract the crop (minimum 300 × 300 px, shifted to fit within the image).
3. Build a feathered soft mask with 8 px falloff.
4. Resize the crop to 512 × 512 and send it to LaMa via ONNX Runtime.
5. Composite the inpainted 512 × 512 result back into the full image using the soft mask.

### SynthID disruption

Applied to both engines, always:
1. Gaussian noise (σ = 1.5) added per-pixel via Box-Muller transform — disrupts LSB steganography.
2. Internal JPEG round-trip at quality 88 — destroys frequency-domain (DCT) fingerprints.
3. Final re-encode to PNG.
